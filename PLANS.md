# Plan Template

## Title

Short name of the change.

## Goal

Describe the user-visible capability and the operational reason for it.

## Scope

List in-scope modules, public artifacts, and configuration changes.

## Data and safety

State data sources, fallback behavior, public/private boundaries, and verification checks.

## Implementation steps

1. Schema/config changes
2. Collector or normalizer changes
3. Scoring/reporting changes
4. Site rendering changes
5. Workflow or deployment changes

## Tests

List unit tests, CLI checks, fixture coverage, and failure scenarios.

## Done when

Define concrete commands and generated files that prove completion.
